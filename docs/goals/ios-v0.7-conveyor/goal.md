# Foil iOS v0.7 GoalBuddy Conveyor

## Original Request

Plan out the next several GoalBuddy prep boards and manage yourself through them.

## Outcome

Run the next Foil iOS work as a sequenced conveyor of focused GoalBuddy boards:

1. Restore physical iPhone automation and prove build 7 on-device.
2. Prove useful dictation quality rather than punctuation-only plumbing.
3. Expand the target-app insertion matrix with sanitized physical evidence.
4. Prepare the next TestFlight feedback/release-readiness loop.

## Oracle

The conveyor is true when each child board has either a merged PR and receipt-backed proof, or an exact blocker receipt with the next safe board/task queued.

## Constraints

- Keep PRs targeting `codex/ios-keyboard-prototype` unless explicitly changing release strategy.
- Do not print or commit API keys, JWTs, private phone content, raw WDA trees from private apps, or transcript bodies from non-sterile contexts.
- Simulator proof supports but cannot replace physical proof for keyboard, microphone, and cross-app insertion.
- Record the strongest realistic failure mode for each board before marking it complete.
