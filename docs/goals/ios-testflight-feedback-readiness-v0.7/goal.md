# Foil iOS TestFlight Feedback Readiness v0.7

## Original Request

Continue toward an early TestFlight loop that testers can actually use and report on.

## Outcome

Prepare the next TestFlight-facing handoff after physical smoke, useful dictation, and insertion coverage: release notes, issue updates, feedback prompts/runbook, and an honest build decision.

## Oracle

The board is true when the next TestFlight handoff tells testers what is proven, what is expected to be clunky, how to recover, and what evidence/feedback we need next.

## Constraints

- Do not upload another build unless the proof state supports it.
- Do not hide physical blockers.
- Keep tester instructions concrete and short.
